# 部署规范

## ⚠️ 核心原则

**未经主人明确确认，严禁自行执行以下操作：**

- ❌ `npm run build` 构建插件
- ❌ 复制 dist 文件到插件目录
- ❌ 重启 gateway（任何方式，包括 pkill、gateway restart、openclaw gateway stop/start）
- ❌ 替换插件文件
- ❌ 任何会导致 gateway 进程中断的操作

## 正确流程

1. 修改代码
2. `git add . && git commit && git push`
3. 向主人汇报改动内容，附上 commit hash
4. **等待主人确认**后，才能执行构建和部署
5. 构建部署后，再次确认是否需要重启 gateway

## 为什么

- 未经确认的部署可能导致线上服务异常
- 主人需要评估风险后才能决定是否部署
- 重启 gateway 会中断当前所有会话
- 插件问题可能影响所有用户

## Git 提交规范

- commit 信息用中文（专业术语除外）
- 增强功能：`feat: <描述>`
- Bug 修复：`fix: <描述>`
- 文档更新：`docs: <描述>`
- 每个独立改动单独提交，不混入无关修改

---

_违反此规范 = 严重错误。_
